# Productivity Skeptic Memory

## Durable Lessons

- Website updates should wait until the Model Vetting Studio or UI-XR runtime can show visible, meaningful video or screenshot evidence from generated humanoids, not just schema, reports, or fixture-grade placeholders.

## Active Risks

- Fixture-grade GLBs can create a false sense of progress if they are not grounded in Anny-compatible generation paths and do not visibly improve toward realistic adaptable humanoids.
- Model-pipeline work can drift into validators, reports, and cagematch scaffolding without producing actual model or runtime improvements.
- Public marketing updates can damage credibility if they show incremental internal plumbing rather than skeptic-impressive model evidence.

## Open Questions

- What is the smallest local model-generation step that makes the next Model Vetting Studio video visibly more credible without paid/cloud providers or restrictive licenses?
- Which Anny-grounded source-quality blocker should be cleared first: case-parameter-to-forward-pass binding, source/weights manifest, or higher-fidelity local source mesh generation?

## Preferred Review Heuristics

- Ask what changed in the GLB, rig, material, animation, lip-sync, gaze, emotion, or runtime behavior.
- Ask whether the Model Vetting Studio has fresh fixed-camera or video evidence after sizeable changes.
- Treat fixture-grade stand-ins as temporary scaffolds only when they directly unblock generated-model fidelity.
- Recommend silence on the public website until a skeptical external viewer would understand the progress without reading internal docs.
